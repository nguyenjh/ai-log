---

## Commit Tagging Quick Reference

Add a tag at the end of your commit message subject line:

| Tag | Meaning |
|-----|---------|
| `[ai:gen]` | AI generated the code, you reviewed and accepted (with or without edits) |
| `[ai:assist]` | You wrote the code, AI helped debug, explain, or improve it |
| `[ai:doc]` | AI helped write or improve documentation, comments, or markdown |
| `[ai:test]` | AI helped write or improve tests |
| `[ai:plan]` | AI was used for planning, requirements thinking, or architecture discussion |
| `[ai:skip]` | No AI used in this commit |

**Examples:**
```
Add note search functionality [ai:gen]
Fix tag filter bug — off-by-one in array slice [ai:assist]
Write ARCHITECTURE.md decision section [ai:doc]
Add unit tests for NoteStore [ai:test]
```

> You do not need to tag every single commit if you are unsure. When in doubt, use `[ai:assist]`.
> The goal is honest signal, not perfect accounting.
